# BioCSSwitch v0.3.3 — Biomedical research packs on CSSwitch v0.3.2

BioCSSwitch v0.3.3 is the first biomedical-focused release for `HERRY423/BioCSSwitch`. It is based on upstream CSSwitch v0.3.2, including real model-name display in Claude Science, relay model guards, and Kimi/MiniMax support, then layers on biomedical MCP/Skill packs for literature, clinical trials, evidence audit, privacy, benchmarks, single-cell analysis, and scFM workflows.

## Highlights

- Rebranded the desktop app and public release/issue/update links for `HERRY423/BioCSSwitch`.
- Added `bio-sc-downstream` for single-cell DEG, trajectory/RNA velocity, communication, marker, and enrichment recipes.
- Added `bio-sc-atlas` for lightweight CELLxGENE dataset search and download skeletons.
- Added `sc-analysis` workflow routing plus `sc_scanpy_pipeline.py` for scanpy end-to-end script generation.
- Expanded `bio-singlecell` with doublet detection, batch integration, gene ID conversion, cell type annotation, multimodal, and spatial transcriptomics recipes.
- Expanded `bio-scfm` with fine-tuning skeletons, embedding quality metrics, and CellFM/UCE preprocessing support.
- Expanded `bio_eval` to 16 categories and 70 cases, including single-cell preprocessing, embedding, DEG, annotation, and safety tests.

## Build Note

This source release was prepared on Windows. The local machine did not have WSL/bash or Rust `cargo`, so the Tauri/Rust build gate and `.dmg` packaging were not completed locally. Before attaching a macOS binary, run:

```bash
bash scripts/release-verify.sh --build
```

on a macOS machine with Rust, Node, npm, and the Tauri toolchain installed.

The formal source-release waiver is recorded in `docs/release-waivers/v0.3.3.md`. It documents the deferred Tauri build, real Claude Science canary, provider matrix, and strict expert-gold approval gates.

## Compatibility

The internal proxy script names, `CSSWITCH_*` environment variables, and `~/.csswitch` config directory are intentionally kept for compatibility with the existing CSSwitch runtime and user configuration. The app bundle identity is changed to `com.biocsswitch.menubar` so BioCSSwitch can be installed separately from upstream CSSwitch.
